# Phase 1: 映像の基礎概念

---

## 1-1. 映像の仕組み

### フレームとフレームレート

動画の正体は**静止画（フレーム）の高速連続表示**。パラパラ漫画と同じ原理。

1秒間に何枚のフレームを表示するかを **フレームレート（fps = frames per second）** と呼ぶ。

| fps | 用途 | 備考 |
|-----|------|------|
| 24fps | 映画 | フィルムの歴史的標準。「映画っぽい」質感 |
| 25fps | 欧州テレビ（PAL） | 50Hzの電源周波数に由来 |
| 30fps（正確には29.97） | 日米テレビ（NTSC） | 60Hzの電源周波数に由来 |
| 60fps | ゲーム、スポーツ中継 | 動きが滑らか。データ量は30fpsの2倍 |

**29.97fps という半端な数字の理由:** NTSCがカラー化されたとき、色信号との干渉を避けるために30fpsから0.1%落とした。歴史的経緯で今も残っている。MediaConvertの設定で `29.97` が出てきたら「NTSCのことね」と思えばよい。

### インターレース vs プログレッシブ

フレームの描画方法に2種類ある。

**プログレッシブ（p）:**
```
フレーム1: 全走査線を一度に描画
  行1 ████████████
  行2 ████████████
  行3 ████████████
  行4 ████████████
  ...
```

**インターレース（i）:**
```
フィールド1（奇数行）:     フィールド2（偶数行）:
  行1 ████████████         行1
  行2                      行2 ████████████
  行3 ████████████         行3
  行4                      行4 ████████████
  ...                      ...
```

- **インターレース** は1フレームを奇数行・偶数行の2回（2フィールド）に分けて描画する。ブラウン管時代の技術で、帯域の半分で動きの滑らかさを確保できた
- **プログレッシブ** は1フレームを丸ごと描画する。現代の液晶/有機ELはこちらが標準

**1080i vs 1080p:**
- 1080i = 1920×1080、インターレース（地デジ放送はこれ）
- 1080p = 1920×1080、プログレッシブ（YouTube、Blu-ray等）
- 同じ1080でも描画方式が違う。1080iの素材を1080pに変換することを **デインターレース** と呼ぶ

> **MediaConvertとの関連:** 入力がインターレース素材（放送系に多い）の場合、デインターレースのフィルタ設定が必要になる。「なぜフィルタが要るのか」はこの知識がないとわからない。

### 解像度

解像度は **横ピクセル数 × 縦ピクセル数** で表す。

| 通称 | 解像度 | ピクセル数 |
|------|--------|-----------|
| SD（480p） | 720×480 | 約35万 |
| HD（720p） | 1280×720 | 約92万 |
| Full HD（1080p） | 1920×1080 | 約207万 |
| 4K（2160p） | 3840×2160 | 約829万 |
| 8K（4320p） | 7680×4320 | 約3,318万 |

4Kは Full HD の4倍のピクセル数 → エンコード時間もデータ量もざっくり4倍になる。

### アスペクト比

映像の縦横の比率。紛らわしい概念が2つある。

**DAR（Display Aspect Ratio）= 画面に表示される見た目の比率**
- 16:9（ワイド）— 現代の標準
- 4:3 — 昔のテレビ
- 21:9（シネスコ）— 映画

**SAR（Sample Aspect Ratio）= 1ピクセルの形の比率**
- 正方形ピクセル → SAR 1:1（PCモニタ、デジタル映像の標準）
- 長方形ピクセル → SAR 1:1以外（DVDなどアナログ由来の素材）

```
実際の関係:
  DAR = SAR × (横ピクセル数 / 縦ピクセル数)

例: DVDの場合
  720×480 で 16:9 表示
  → SAR = 16/9 ÷ (720/480) = 32/27 ≈ 1.185
  → ピクセルが横に少し長い
```

> **MediaConvertとの関連:** 古い素材や放送素材では SAR が1:1でないことがある。MediaConvertで解像度を変換するとき、SARを考慮しないと映像が引き伸ばされたり潰れたりする。設定に「PAR（Pixel Aspect Ratio）」という項目が出てきたら、これはSARと同義。

## 1-2. 色と画質の基礎

### 色空間

映像が「どの範囲の色を表現できるか」を定めた規格。

| 規格 | 用途 | 色の範囲 |
|------|------|---------|
| BT.601 | SD映像 | 狭い |
| BT.709 | HD映像（現在の主流） | 標準 |
| BT.2020 | 4K/8K、HDR映像 | 広い |

プログラミングでいう型のようなもの。同じRGB値(200, 50, 50)でも、BT.709とBT.2020では異なる色になる。変換時に色空間の情報を正しく引き継がないと色味が変わる。

### SDR と HDR

**SDR（Standard Dynamic Range）:**
- 従来の映像。明るさの表現範囲が狭い
- 最大輝度 ~100 nits（ニト）

**HDR（High Dynamic Range）:**
- 明るさの表現範囲が広い。太陽の眩しさや暗闇の階調を表現できる
- 最大輝度 1,000〜10,000 nits

HDRの主要規格:

| 規格 | 特徴 |
|------|------|
| HDR10 | 静的メタデータ（映像全体で1つの明るさ設定）。オープン標準、最も普及 |
| HDR10+ | 動的メタデータ（シーンごとに明るさ設定）。Samsung主導 |
| HLG | メタデータ不要。SDRディスプレイでもそれなりに映る。放送向き |
| Dolby Vision | 動的メタデータ。最高品質だがライセンス必要 |

> **MediaConvertとの関連:** HDR素材を扱う場合、色空間（BT.2020）、転送特性（PQ / HLG）、メタデータの設定が必要になる。SDR→HDR変換（トーンマッピング）もMediaConvertで可能。

### ビット深度

1ピクセルの各色チャンネル（R, G, B）を何ビットで表現するか。

| ビット深度 | 階調数（1チャンネルあたり） | 総色数 |
|-----------|--------------------------|--------|
| 8bit | 256段階 | 約1,677万色 |
| 10bit | 1,024段階 | 約10億色 |
| 12bit | 4,096段階 | 約687億色 |

8bitと10bitの差は、特に**グラデーション**で顕著に出る。

```
8bit のグラデーション:
  ██ ██ ██ ██ ██ ██ ██ ██  ← 段差が見える（バンディング）

10bit のグラデーション:
  ████████████████████████  ← 滑らか
```

- SDR映像 → 8bit が一般的
- HDR映像 → 10bit 以上が必須（明るさの表現範囲が広いのに階調が少ないとバンディングが目立つ）

> **MediaConvertとの関連:** コーデックによって対応ビット深度が異なる。H.264は基本8bit（10bitも可能だが互換性低い）、H.265/AV1は10bit対応が標準。出力設定でビット深度を選ぶとき、ここの知識が判断基準になる。

## 1-3. 音声の基礎

### サンプリングレートとビット深度（音声）

音声はアナログの波形をデジタル化（サンプリング）して記録する。

**サンプリングレート** = 1秒間に何回波形を計測するか
- 44.1kHz — CD品質（44,100回/秒）
- 48kHz — **映像の標準**（DVD、Blu-ray、配信すべてこれ）
- 96kHz / 192kHz — ハイレゾ音源

> 映像を扱うなら基本 **48kHz** と覚えておけばよい。44.1kHzは音楽CDの規格。

**ビット深度（音声）** = 1サンプルの精度
- 16bit — CD品質、十分な品質
- 24bit — プロ制作標準
- 32bit float — 制作中の中間フォーマット

### チャンネル構成

| 構成 | チャンネル数 | 用途 |
|------|------------|------|
| モノラル | 1 | 通話、ポッドキャスト |
| ステレオ | 2（L, R） | 音楽、一般的な動画 |
| 5.1ch | 6（FL, FR, C, LFE, SL, SR） | 映画、Blu-ray |
| 7.1ch | 8 | ホームシアター、劇場 |

```
5.1ch の配置:
        C（センター）
  FL ──────── FR
（前左）      （前右）
       🧑 視聴者
  SL ──────── SR
（後左）      （後右）
     LFE（サブウーファー = 重低音 = ".1"の部分）
```

### ラウドネス（LUFS）

音の「聞こえる大きさ」の国際基準。

テレビでCMだけ音量が大きくて不快になった経験があるかもしれない。あれを防ぐために、放送・配信では音量の基準が定められている。

| 規格 | 基準値 | 用途 |
|------|--------|------|
| EBU R128 | -23 LUFS | 欧州放送 |
| ATSC A/85 | -24 LKFS（≒LUFS） | 北米放送 |
| 配信系 | -14〜-16 LUFS | YouTube、Spotify等 |

> **MediaConvertとの関連:** MediaConvertには「Audio normalization（音声正規化）」の設定がある。ラウドネスの基準値を指定して、入力素材の音量を自動で調整してくれる。放送納品や配信で必須の機能。
