# 02-types-and-functions Q&A

2026-03-07

## Q1: `S (S n') => evenb n'` の `n'` とは何か

`n'` はパターンマッチで取り出された変数。名前は自由で、`n'` は「nから派生した値」の慣習的命名。

`S (S n')` は「nから S を2つ剥がして残りを n' に束縛する」パターン。具体例:

```
evenb 4
= evenb (S (S (S (S O))))
→ S (S n') にマッチ、n' = S (S O) = 2
→ evenb 2
→ S (S n') にマッチ、n' = O = 0
→ evenb 0
→ O にマッチ → true
```

つまり「2ずつ引いていって、0になれば偶数、1になれば奇数」。プログラミングの `return evenb(n - 2)` に相当。

## Q2: なぜ再帰関数だけ Fixpoint なのか。Definition ではだめなのか

`Definition` は「名前に式を束縛する」だけで、定義の中で自分自身を参照できない。再帰関数を書くとエラーになる。

`Fixpoint` は「再帰的な定義である」とRocqに伝え、さらに引数が構造的に小さくなっているか（＝必ず停止するか）を自動チェックする。

Rocqではすべてのプログラムの停止が保証される必要があるため、再帰は明示的に `Fixpoint` と宣言する。普通のプログラミング言語では関数がデフォルトで再帰できるので区別がないが、Rocqでは意図的に分けている。

## Q3: `myplus` の定義がわからない

`myplus n m` は「nから S を1つずつ剥がし、結果の外側に S を1つずつ積む。nが0になったらmを返す」。

```
myplus 3 2
→ S (myplus 2 2)     (* n=3, S n' にマッチ、n'=2 *)
→ S (S (myplus 1 2))   (* n=2, S n' にマッチ、n'=1 *)
→ S (S (S (myplus 0 2)))  (* n=1, S n' にマッチ、n'=0 *)
→ S (S (S 2))          (* n=0, O にマッチ → m=2 *)
= 5
```

つまり `n + m` は「mの上にSをn個積む」操作。左側(n)だけ分解し、右側(m)はそのまま。

## Q4: `mymult` の定義

`mymult n m` は「m を n 回足す」。myplus が「Sをn個積む」だったのと同じ発想で、mymult は「myplus を n 回繰り返す」。

```
mymult 3 4
→ myplus 4 (mymult 2 4)           = 4 + (2*4)
→ myplus 4 (myplus 4 (mymult 1 4))     = 4 + 4 + (1*4)
→ myplus 4 (myplus 4 (myplus 4 (mymult 0 4))) = 4 + 4 + 4 + (0*4)
→ myplus 4 (myplus 4 (myplus 4 O))       = 4 + 4 + 4 + 0
= 12
```

## Q5: ペアノ数の計算は大きい数で遅くならないのか

遅い。`Compute (10000 * 10000).` とかは固まる。ただし問題にならない。

- **Rocq内**: ペアノ数は構造がシンプルで証明しやすいから使う。効率は重要でない
- **実行時**: Extraction でOCaml等に変換すると普通のネイティブ整数になる

証明のための表現（シンプルさ重視）と実行のための表現（効率重視）を分けるのがRocqの設計思想。

## Q6: 四則演算すべてのペアノ数での定義

| 演算 | 考え方 |
|------|--------|
| `n + m` | nからSを1つずつ剥がし、結果にSを積む。nが0になったらmを返す |
| `n * m` | mをn回足す（myplusのn回繰り返し） |
| `n - m` | 両方からSを1つずつ剥がす。自然数に負の数がないので `3 - 5 = 0` |
| `n / m` | nからmを繰り返し引く。引けた回数が商 |

全部「Sを積む/剥がす」の繰り返しで定義される。

## Q7: factorial の定義の読み方

`S n'` にマッチすると `n'` は `n-1`、`S n'` は `n` そのもの。なので `S n' * factorial n'` = `n * factorial(n-1)`。ペアノ数だと `n` と `n-1` が同時に手に入る。

```
factorial 4 = 4 * factorial 3 = 4 * 3 * factorial 2 = ... = 4 * 3 * 2 * 1 * 1 = 24
```
