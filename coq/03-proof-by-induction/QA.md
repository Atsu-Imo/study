# 03-proof-by-induction Q&A

2026-03-07

## Q1: `induction n as [| n' IHn']` の `|` とは何か。`IHn'` は慣習の命名か

`[| n' IHn']` は帰納法の各ケースで使う変数名を指定する構文。`|` はケースの区切り。

- `|` の左（空）: O のケース。パラメータがないので名前なし
- `|` の右: S のケース。`n'`（S の中身）と `IHn'`（帰納法の仮定）

`IHn'` は Induction Hypothesis for n' の略で慣習的命名。好きな名前にできる。`as [...]` は省略可能だが明示が推奨。`destruct` でも同じ構文が使える（ただし帰納法の仮定はない）。

## Q2: plus_assoc の帰納ステップで simpl するとなぜゴールに近づくのか

`+` の定義が `S n' + m → S (n' + m)` なので、`simpl` はこれを適用しているだけ。

```
(S a' + b) + c = S a' + (b + c)
  ↓ simpl
S ((a' + b) + c) = S (a' + (b + c))
```

両辺の外側の S が同じなので、中身 `(a' + b) + c = a' + (b + c)` が言えればよく、これが帰納法の仮定 IHa' そのもの。`simpl` で S が外に出ることで IH が使える形になるのが、帰納法 + simpl + rewrite IH パターンの仕組み。
