# Rocq学習プロジェクト — Claude向け指示書

## プロジェクト概要

このディレクトリはソフトウェア検証を目的としてRocq (旧Coq) を学ぶためのカリキュラムである。カリキュラムの詳細は `CURRICULUM.md` を参照すること。

## 環境

- Rocq 9.0.1 (Rocq Platform 2025.08 経由でWSL2にインストール)
- IDE: RocqIDE (`rocqide`)
- コンパイル: `rocq c` (旧 `coqc`)

## 学習の進め方

- ユーザーが次のトピックに進みたいと言ったら、`CURRICULUM.md` を参照して次のトピックのディレクトリを作成する
- 各トピックは独立したディレクトリとして作成する
- 既存トピックのフォーマットと構成を踏襲する (`01-first-proof` を参考にすること)

## コーディングスタイル

- コメントは日本語で記述する
- Rocqの慣例に従う (型名は大文字始まり、定理名は snake_case)
- `.v` ファイル内のコードは教育目的であることを意識し、段階的に概念を導入する
- 各 `.v` ファイルは `rocq c` でコンパイル可能な状態を維持する

## README.md の記述ルール

- 日本語で記述する
- プログラミング経験者の視点を意識し、馴染みのある概念との比較を含める
- コードブロックには言語指定をつける (```coq)
- 演習は3段階 (基礎 → 応用 → チャレンジ) で構成する

## Q&A記録ルール

- ユーザーが学習中に質問してきたら、回答後にそのQ&Aを `XX-topic-name/QA.md` に追記する
- フォーマット:
  ```
  ## QN: (質問内容)
  (回答内容)
  ```
- 質問番号は通し番号 (Q1, Q2, ...)
- 日付はQ&Aファイルの先頭に記録する

## トピック作成時のチェックリスト

1. `XX-topic-name/main.v` を作成 (コンパイル可能な状態)
2. `XX-topic-name/README.md` を作成 (概念説明 + 演習)
3. `rocq c main.v` で正常にコンパイルできることを確認
