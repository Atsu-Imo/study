## 🔒 Vulnerabilities & Security
- **[shell-quote: `quote()` does not escape newlines in object `.op` values](https://github.com/advisories/GHSA-w7jw-789q-3m8p)** — `CVE-2026-9277 / Critical / npm`
  - 広く使われる shell-quote の `quote()` がオブジェクトの `.op` 値内の改行をエスケープせず、コマンドインジェクションに繋がりうる（6/9）。transitive 依存に入りがちなパッケージなので、`npm ls shell-quote` で依存元を確認し最新化を。
- **[Dex (dexidp): token-exchange endpoint missing `AllowedConnectors` enforcement](https://github.com/advisories/GHSA-7qjx-gp9h-65qj)** — `High / CVSS 8.7 / Go`
  - OIDC プロバイダ Dex の `handleTokenExchange` が、他のハンドラには入っている `isConnectorAllowed()` チェックを**書き忘れ**ており、漏洩したクライアントシークレットで本来許可されないコネクタのトークンを交換できる（6/9）。**ただし影響は master の未リリースコードのみで、安定版 v2.45.1 は非該当**。とはいえ「兄弟ハンドラには入れた認可チェックを片方だけ落とす」という Go の認可設計の典型ミスで、昨日の nebula-mesh IDOR と並ぶ良い教材。
- **[symfony/runtime: `APP_ENV`/`APP_DEBUG` override via `parse_str` (CVE-2024-50340 patch bypass)](https://github.com/advisories/GHSA-fqc7-9xjw-jrh3)** — `CVE-2026-47767 / Moderate / PHP`
  - 過去の CVE-2024-50340 の修正をすり抜け、クエリ経由で `APP_ENV`/`APP_DEBUG` を書き換えられる回帰（6/9）。「パッチバイパス」は再発しやすいパターンの好例。Symfony 利用環境は修正版へ。
- **[phoenix_storybook: Unauthenticated RCE via HEEx template injection](https://github.com/advisories/GHSA-55hg-8qxv-qj4p)** — `CVE-2026-8467 / Critical / Elixir`
  - Phoenix Storybook で HEEx テンプレートインジェクションによる無認証 RCE。同時に atom 無制限生成による DoS（CVE-2026-8469）等のクラスタあり（6/9）。Elixir/Phoenix を使うなら優先対応。
