- **[CISA KEV: BerriAI LiteLLM Command Injection added (actively exploited)](https://www.cisa.gov/news-events/alerts/2026/06/08/cisa-adds-two-known-exploited-vulnerabilities-catalog)** — `CVE-2026-42271 / KEV 追加 6/8`

- **[CISA KEV: 6/8–6/9 で計5件追加（前回 6/3 以来の更新）](https://www.cisa.gov/news-events/alerts/2026/06/09/cisa-adds-three-known-exploited-vulnerabilities-catalog)** — `Exploited`

- **[OpenTelemetry Operator: ServiceMonitor `bearerTokenFile` reads arbitrary local file](https://github.com/advisories)** — `CVE-2026-47701 / High / Go`
  - OTel Operator が `bearerTokenFile` で**任意のローカルファイルを読み、その中身を bearer 認証として外部送信**してしまう。K8s 上でサービストークンや秘密が抜ける典型。「設定で指すファイルパスを信頼してしまう」Go の設定設計ミスとして教材性も高い。OTel Operator 利用クラスタは要確認。
- **[go-base: Hardcoded JWT Secret 'random' allows token forgery](https://github.com/advisories)** — `CVE-2026-48031 / Critical / Go`
  - スターターテンプレート go-base が JWT 署名鍵を文字列 `"random"` でハードコードしており、**誰でも任意トークンを偽造可能**。テンプレ/ボイラープレートをそのまま本番に持ち込む危険の好例。Go の認証実装レビュー観点として一読の価値あり。
- **[RUSTSEC-2026-0175 — `onering` 1.4.1 removed from crates.io (malicious code)](https://rustsec.org/advisories/RUSTSEC-2026-0175.html)** — `Malicious / Rust`
