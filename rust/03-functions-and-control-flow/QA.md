# Q&A: 03-functions-and-control-flow

日付: 2026-03-06

## Q1: if式の有用さがわからない。どんなケースで有効？

最大のメリットは「**変数の初期化と条件分岐を1つの式にまとめ、変数を不変に保てる**」こと。

Goでは条件で初期値を変えるとき `var msg string` + if/else で代入するので変数が可変になる。Rustでは:
```rust
let msg = if user.is_admin {
    "管理画面へようこそ"
} else {
    "一般画面へようこそ"
};
// msg は不変。以降の誤った再代入を防げる
```

複雑な条件でも1つの `let` で完結:
```rust
let discount = if is_member && total > 10000 {
    0.2
} else if is_member {
    0.1
} else {
    0.0
};
```

Rustの「不変がデフォルト」という設計と噛み合った機能。match式（07で学ぶ）と組み合わせるとさらに強力。

## Q2: if式の結果を引数で受け取って関数から返すこともできる？

できる。関数の本体がif式1つだけなら、return も let も不要:
```rust
fn welcome_message(is_admin: bool) -> &'static str {
    if is_admin {
        "管理画面へようこそ"
    } else {
        "一般画面へようこそ"
    }
}
```

「関数自体が式を返す」のと「ifが式である」のが組み合わさった形。`&'static str` のライフタイム注釈は `11-lifetimes` で学ぶ。今は「文字列リテラルの型」と理解すればOK。

## Q3: loopがbreak値を返せるのはなぜ？評価のタイミングによるもの？

根本的な理由は**Rustでは「ほぼ全てが式」だから**。

- 文（statement）→ 値を返さない。`let x = 5;` など
- 式（expression）→ 値を返す

`if`、`loop`、`match`、ブロック `{}` が全て式:
```rust
let x = { let a = 1; let b = 2; a + b };  // x = 3
let result = loop { break 42; };           // result = 42
```

セミコロンの意味もここに繋がる。セミコロンは「式を文に変換する」役割:
```rust
let a = { 5 };    // a = 5（式）
let b = { 5; };   // b = ()（セミコロンで文になり値が捨てられる）
```

Goの `for`/`if` は「文」として設計されているので値を返せない。Rustは関数型言語（ML系、Haskell等）の影響を受けた「式ベース」の言語設計。

## Q4: 逆に式ではないもの（文のみ）は何？

Rustで文のみのものは少ない:
- `let` 宣言 — `let x = 5;` は値を返さない
- `fn` / `struct` / `enum` 等の定義（アイテム定義）
- `use` / `mod`（モジュール関連の宣言）
- セミコロン付きの式 — `x + 1;` で文に変換される

```rust
// let は文なので値を返せない
// let x = (let y = 5);  // コンパイルエラー
```

「何かを宣言・定義するもの」が文、「値を計算・生成するもの」が式、と覚えればほぼ正しい。

## Q5: 演習3の意図と `sum + 1` のバグについて

`sum + 1;` は式の結果を捨てているだけで `sum` は変わらない（無限ループになる）。`sum += 1;` が正しい。

演習の意図は「1+2+3+4+...と順に足して合計が100を超えた時のiを返す」:
```rust
let mut sum = 0;
let mut i = 0;
let n = loop {
    i += 1;
    sum += i;
    if sum > 100 {
        break i;
    }
};
// → 14の時に合計が100を超えた（合計: 105）
```
