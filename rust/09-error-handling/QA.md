# 09 - エラーハンドリング Q&A

日付: 2026-03-10

## Q1: エラーを返しうる関数では基本的に Result<T, E> を返すのがベスト？

はい。Goで `(T, error)` を返すのと同じ感覚で、回復可能なエラーには `Result<T, E>` を使うのが基本。

ただし例外的なケースもある:

| パターン | 使い分け |
|---|---|
| `Result<T, E>` | **基本はこれ**。回復可能なエラー |
| `Option<T>` | エラーではなく「値がないかもしれない」場合（`map.get()` など） |
| `panic!` | バグ・回復不能な状態（配列の範囲外アクセスなど） |

Goだと「値がない」も「エラー」も全部 `(T, error)` で返しがちだが、Rustは `Option` と `Result` を使い分ける。「なぜ失敗したか」の情報が要るなら `Result`、要らないなら `Option`。
