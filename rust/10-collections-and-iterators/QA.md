# 10 - コレクションとイテレータ Q&A

日付: 2026-05-20

## Q1: 動的配列ってどういうケースで使うんだろう？Rustのコンパイル時に決まっているという原則から離れているように見える

Rustの「コンパイル時に決まる」は**値**ではなく**型・所有権・借用**の話。`Vec<i32>` も「i32 を任意個入れる動的配列」という型として固定されており、型システムから外れてはいない。

ただし、Rust には「サイズも静的」な配列が別にある:

```rust
let arr: [i32; 3] = [1, 2, 3];     // 固定長配列。長さが型の一部！スタック上
let vec: Vec<i32> = vec![1, 2, 3]; // 動的配列。ヒープ上
```

`[i32; 3]` と `[i32; 4]` は**別の型**になる（重要）。

### 使い分け

**`[T; N]`（固定長配列）を使うケース** — サイズが意味的に固定:
- 暗号ハッシュ: `[u8; 32]` (SHA-256)
- IPアドレス: `[u8; 4]` (IPv4)
- 色: `[u8; 3]` (RGB)
- 行列: `[[f32; 4]; 4]`
- 組み込み環境（ヒープアロケータなし）

型レベルで `[u8; 32]` と `[u8; 16]` を区別できるので、「ハッシュ長を間違えた」をコンパイラが検出してくれる。

**`Vec<T>` を使うケース** — 長さが実行時に変わる:
- ユーザー入力など長さが不明
- 動的に追加・削除したい
- 関数の戻り値で「いくつ返るか不定」

### Go との違い

Goは `[N]int` をほぼ使わず `[]int`（slice）ばかり使うので「配列＝動的」が刷り込まれている。
Rustは型システムが強いぶん `[T; N]` を積極的に使う。Vec は「長さを実行時に変えたい時のためのヒープデータ構造」であり、`[T; N]` の上位互換ではなく用途違い。

## Q2: `v[0]` がパニックするの？1が入ってるように見えるんだけど

パニックしない。`v[0]` は普通に `1` を返す。README/main.rsのコメントが誤解を招く書き方だった（修正済み）。

正確には:

```rust
let v = vec![1, 2, 3];
v[0]      // => 1       (OK)
v[3]      // => panic!  (index out of bounds)
v.get(0)  // => Some(&1)
v.get(3)  // => None
```

| アクセス方法 | 範囲内 | 範囲外 |
|---|---|---|
| `v[i]` | 値を返す | **パニック** |
| `v.get(i)` | `Some(&値)` | `None`（パニックしない） |

Go の `s[i]` と Rust の `v[i]` は同じ挙動（範囲外でランタイムパニック）。違いは Rust に `v.get(i)` という安全な代替があること。Goには相当するものがなく、自前で `if i < len(s)` チェックが必要。

## Q3: `v.push(x)` って `v` の所有権を奪うのでは？

奪わない。`push` のシグネチャは `fn push(&mut self, value: T)` で、`&mut self` は「変更可能な借用」であり所有権の移譲ではない。

**「変更する」と「所有権を奪う」は別概念**。これは Rust 固有で、Go には対応する概念がない（値レシーバ/ポインタレシーバの区別はあるが、所有権という概念自体がない）。

### self の3段階

| 形式 | 意味 | 呼び出し後の `v` |
|---|---|---|
| `&self` | 不変借用 | そのまま使える |
| `&mut self` | 可変借用 | **そのまま使える**（中身は変わる） |
| `self` | 所有権を奪う | **使えなくなる** |

### 具体例

```rust
let mut v = vec![1, 2, 3];
v.push(4);
v.push(5);
println!("{:?}", v);  // [1, 2, 3, 4, 5] — v はそのまま使える
```

もし `push` が所有権を奪うなら2回目の呼び出しで「moved value」エラーになる。実際はならない。

### 「奪う」メソッドの慣習

`into_xxx` という名前は「所有権を取って別の型に変換する」サイン:

```rust
let v = vec![1, 2, 3];
let it = v.into_iter();  // v の所有権が it へ移動
// println!("{:?}", v);  // コンパイルエラー
```

### `push(x)` は `x` の所有権は奪う

`v` は借用するだけだが、引数 `x` は `T` 型なので所有権を奪う:

```rust
let mut v: Vec<String> = Vec::new();
let s = String::from("hello");
v.push(s);
// println!("{}", s);  // コンパイルエラー: s は v に移動した
```
