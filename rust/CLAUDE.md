# Rust学習プロジェクト — Claude向け指示書

## プロジェクト概要

このディレクトリはGo経験者がRustを学ぶためのカリキュラムである。カリキュラムの詳細は `CURRICULUM.md` を参照すること。

## 学習の進め方

- ユーザーが次のトピックに進みたいと言ったら、`CURRICULUM.md` を参照して次のトピックのディレクトリを作成する
- 各トピックは独立したCargoプロジェクトとして作成する (`cargo init` は使わず直接ファイルを作成)
- 既存トピックのフォーマットと構成を踏襲する (`01-hello-cargo` を参考にすること)

## コーディングスタイル

- `cargo clippy` の警告をゼロにする
- `cargo fmt` でフォーマット済みの状態を維持する
- コメントは日本語で記述する
- 変数名・関数名はRustの慣例に従う (snake_case)
- `main.rs` 内のコードは教育目的であることを意識し、段階的に概念を導入する
- `main.rs` の末尾に演習用のコード（雛形・ヒント付き）を含める。コンパイルが通るようにコメントアウトまたは未呼び出しの状態にする

## README.md の記述ルール

- 日本語で記述する
- Go経験者の視点を常に意識し、Goとの比較を含める
- コードブロックには言語指定をつける (```rust, ```go)
- 演習は3段階 (基礎 → 応用 → チャレンジ) で構成する

## Q&A記録ルール

- ユーザーが学習中に質問してきたら、回答後にそのQ&Aを `XX-topic-name/QA.md` に追記する
- フォーマット:
  ```
  ## QN: (質問内容)
  (回答内容)
  ```
- 質問番号は通し番号 (Q1, Q2, ...)
- 日付はQ&Aファイルの先頭に記録する

## トピック作成時のチェックリスト

1. `XX-topic-name/Cargo.toml` を作成
2. `XX-topic-name/src/main.rs` を作成 (コンパイル・実行可能な状態)
3. `XX-topic-name/README.md` を作成 (概念説明 + Go比較 + 演習)
4. `cargo run` で正常に動作することを確認
5. `cargo clippy` で警告がないことを確認
