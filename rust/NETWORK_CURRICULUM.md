# ネットワーク学習カリキュラム — Rustで学ぶネットワークの登場人物たち

## コンセプト

「`https://example.com` にアクセスしたとき、裏で何が起きていて、誰が何をしているのか」を
**各登場人物を実際にRustで実装することで理解する**カリキュラム。

低レイヤーの積み上げではなく、**ネットワーク上の「役者」を1人ずつ作って動かす**ことに重点を置く。

## 全体像 — ブラウザがWebページを表示するまでの登場人物

```
ユーザーが https://example.com にアクセスすると...

  ┌──────────┐
  │ ブラウザ  │
  └────┬─────┘
       │ ① "example.com のIPアドレスは？"
       ▼
  ┌──────────┐
  │ DNSサーバー│  ← net-03 で実装
  └────┬─────┘
       │ "93.184.216.34 だよ"
       ▼
  ┌──────────┐     ┌──────────────┐
  │ ブラウザ  │────>│ ファイアウォール│  ← net-09 で実装
  └────┬─────┘     └──────┬───────┘
       │ ② TLS接続開始      │ 通過OK
       ▼                   ▼
  ┌──────────────────────────┐
  │ TLSハンドシェイク          │  ← net-04 で実装
  │ (証明書検証 → 暗号化確立)  │
  │                          │
  │  ┌─────┐                 │
  │  │ CA  │ 証明書を発行     │  ← net-05 で実装
  │  └─────┘                 │
  └────┬─────────────────────┘
       │ ③ 暗号化されたHTTPSリクエスト
       ▼
  ┌──────────────┐
  │ HTTPS Proxy  │  ← net-06 で実装 (企業ネットワークなどにいる)
  └──────┬───────┘
         │ ④ 転送
         ▼
  ┌──────────────┐
  │ ロードバランサー│  ← net-08 で実装
  └──────┬───────┘
         │ ⑤ バックエンドサーバーを選択
         ▼
  ┌──────────────┐
  │ リバースプロキシ│  ← net-07 で実装 (nginx的な役割)
  └──────┬───────┘
         │ ⑥ パスに応じてルーティング
         ▼
  ┌──────────────┐
  │ Webサーバー   │  ← net-02 で実装
  └──────────────┘
```

## 各トピックの詳細

### net-01: TCPソケット ✅

（完了済み）TCP接続、ハンドシェイク、エコーサーバー

### net-02: Webサーバー

- **登場人物**: Webサーバー（Apache/nginx的な存在）
- **作るもの**: HTTPリクエストをパースしてレスポンスを返す簡易サーバー
- **理解できること**:
  - HTTPは「TCPの上に乗ったテキストプロトコル」にすぎない
  - リクエスト行 `GET /index.html HTTP/1.1` の意味
  - ヘッダー、ステータスコード、Content-Type
  - ブラウザで実際にアクセスできる

### net-03: DNSサーバー & リゾルバ

- **登場人物**: DNSサーバー（ネームサーバー）
- **作るもの**: ドメイン名→IPアドレスの問い合わせに応答するDNSサーバー + 問い合わせるリゾルバ
- **理解できること**:
  - ブラウザがURLを打ったとき、最初に起きるのがDNS問い合わせ
  - DNSメッセージのバイナリフォーマット（ヘッダー・質問・回答セクション）
  - 再帰問い合わせ vs 反復問い合わせ
  - DNSキャッシュの仕組み

### net-04: TLSハンドシェイク

- **登場人物**: TLSプロトコル（HTTPSの「S」）
- **作るもの**: TLSハンドシェイクのシミュレーション + 実際のTLS接続
- **理解できること**:
  - 「HTTPSは暗号化されたHTTP」の具体的な意味
  - ClientHello → ServerHello → 証明書 → 鍵交換 → 暗号化開始
  - 共通鍵暗号 vs 公開鍵暗号の使い分け
  - なぜ最初の鍵交換に公開鍵暗号を使い、データ転送は共通鍵暗号なのか

### net-05: 認証局（CA）

- **登場人物**: 認証局（Certificate Authority）
- **作るもの**: 自己署名CA → サーバー証明書の発行 → 証明書の検証
- **理解できること**:
  - 「この証明書は信頼できる」とは具体的に何を検証しているのか
  - ルートCA → 中間CA → サーバー証明書の信頼チェーン
  - 証明書の中身（Subject, Issuer, 有効期限, 公開鍵, 署名）
  - ブラウザの「鍵マーク」の裏側

### net-06: HTTPS Proxy

- **登場人物**: フォワードプロキシ（企業ネットワーク等で使われる）
- **作るもの**: HTTP CONNECT メソッドでTLSトンネルを中継するプロキシ
- **理解できること**:
  - HTTPプロキシとHTTPSプロキシの根本的な違い
  - CONNECT メソッドによるトンネリング（暗号化を壊さずに中継）
  - 企業プロキシがTLS通信を検査する仕組み（MITM型プロキシ）
  - プロキシ認証（Proxy-Authorization）

### net-07: リバースプロキシ

- **登場人物**: リバースプロキシ（nginx/Caddy的な存在）
- **作るもの**: パスに応じてバックエンドサーバーに振り分けるプロキシ
- **理解できること**:
  - フォワードプロキシ vs リバースプロキシの違い
  - `/api/*` → バックエンドA、`/static/*` → バックエンドB のルーティング
  - `X-Forwarded-For` などのヘッダー書換え
  - TLS終端（クライアント↔リバプロはHTTPS、リバプロ↔バックエンドはHTTP）

### net-08: ロードバランサー

- **登場人物**: ロードバランサー（AWS ALB/NLB的な存在）
- **作るもの**: 複数バックエンドにリクエストを分散するL4/L7ロードバランサー
- **理解できること**:
  - 分散アルゴリズム: ラウンドロビン、重み付き、最少接続数
  - ヘルスチェック（バックエンドが生きているか定期確認）
  - L4 (TCP) vs L7 (HTTP) ロードバランシングの違い
  - スティッキーセッション

### net-09: ファイアウォール

- **登場人物**: ファイアウォール（iptables/nftables的な存在）
- **作るもの**: ルールベースのパケットフィルタ
- **理解できること**:
  - 「ポート80は許可、それ以外は拒否」の実装
  - ルール評価順序（先にマッチしたルールが勝つ）
  - ステートフルファイアウォール（接続追跡）
  - ALLOW/DENY/DROP の違い

### net-10: ルーター & NAT

- **登場人物**: ルーター、NATゲートウェイ
- **作るもの**: IPパケットを宛先に応じて転送するルーター + NATシミュレーション
- **理解できること**:
  - ルーティングテーブル（宛先IP → 次のホップ）
  - NAT: プライベートIP ↔ グローバルIP の変換
  - なぜ家庭のPCがインターネットにアクセスできるのか
  - サブネットとCIDR

### net-11: ネットワークシミュレーター

- **登場人物**: 全員
- **作るもの**: 1つのHTTPSリクエストが全登場人物を通過する様子を可視化
- **理解できること**:
  - 全体の流れを1つのプログラムで体感
  - 各登場人物がどのレイヤーで何をしているか
  - パケットが変換・暗号化・ルーティングされる過程

## 使用クレート方針

| トピック | クレート |
|---|---|
| 01-03 | `std::net` のみ |
| 04-05 | `ring` or `rustls`（暗号処理）, `x509-parser`（証明書パース） |
| 06-08 | `tokio`（非同期I/O）, 必要に応じて `rustls` |
| 09-10 | `pnet`（パケット操作）, シミュレーションは `std` のみも可 |
| 11 | 各トピックの組み合わせ |

## 各トピックの構成

```
net-XX-topic-name/
├── Cargo.toml
├── README.md          ← 登場人物の役割説明 + 図解 + 演習
└── src/main.rs        ← 動くコード + 演習の雛形
```

## 難易度カーブ

- **01-02**: 入門 — TCP/HTTPの基本
- **03**: 中級 — バイナリプロトコルのパース
- **04-05**: 中〜上級 — 暗号の概念が入る（ただし暗号ライブラリを使うので数学は不要）
- **06-08**: 中級 — Part 1-2の知識を組み合わせる
- **09-10**: 上級 — 低レイヤーのパケット操作
- **11**: 統合 — 全体を繋げる

## 進め方

1. 各トピックの README.md でその「登場人物」の役割を把握する
2. `src/main.rs` のコードを読み、`cargo run` で実行する
3. 実際に通信している様子を観察する
4. 演習に取り組んで理解を深める
5. 次の登場人物へ
